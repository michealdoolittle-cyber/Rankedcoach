PROJECT PURPOSE
RankedCoach is a post-match VALORANT coaching and self-review app.
It is intended to help players understand their own match history, logs, habits, agents, maps, weapons, and improvement focus.
The app is private to the signed-in player. It does not evaluate other players.
It does not provide live in-game advantage.
It does not provide MMR, ELO, or an alternative ranked ladder.
It does not claim Riot approval or partnership.
It should use RSO/OAuth once Riot approves production access for player stat display.

RIOT APPLICATION POSITION
The product should be presented as:
- A post-match coaching and reflection tool.
- RSO/OAuth based for player consent.
- Personal/private player data only.
- No player shaming.
- No unofficial rank/MMR/ELO calculator.
- No live game advantage.
- No use of Riot official logos.
- Game art assets are used where appropriate.

Requested APIs for Riot application:
- RSO / OAuth
- Account-V1
- VAL-MATCH-V1
- VAL-CONTENT-V1
- VAL-RANKED-V1, if available/appropriate for rank context

Policy risk reminders:
- Do not charge for exclusive app access while under Riot review.
- Donations/tips can exist, but should not gate functionality.
- Avoid language that shames players.
- Keep all coaching language focused on self-improvement.
- Keep API keys server-side only.

CURRENT PRODUCT DESCRIPTION SUMMARY
RankedCoach helps VALORANT players review their own ranked performance after matches by combining Riot-authorized match data, player reflections, and coaching formulas.
It gives a Home dashboard, Logging page, Stats page, Insights page, and Ask Coach assistant.
It is built for personal improvement, not public comparison.

MFA STATUS
The plan is TOTP-only for now.
Phone/SMS MFA was discussed, but SMS requires a provider or hook and is not free/simple by default.
The current direction is:
- Enable MFA via TOTP.
- Show a QR code when MFA is enabled.
- Let user scan with an authenticator app such as 2FAS.
- Remove email/phone MFA selection for now unless SMS/email MFA is fully implemented later.

SESSION RR / CHART BEHAVIOR
Important correction made on 2026-05-30:
The RR total card and chart should represent the current selected session/day only, not the entire season/act.
The nav rank and profile-level progress can still understand broader profile context, but the Home RR card/chart is a session tracker.
This matters because most users play 3 to 5 games in a session, not 20+.

Session/date selection is tied to the logging calendar current session date.
The chart x-axis should reflect that session window.
The RR match stats panel uses the latest match from the current selected session.

MAP STATS BEHAVIOR
Map cards should be sorted by win rate.
Active maps come first.
Excluded maps are locked, greyed out, and placed at the bottom.
Excluded maps should not be clickable.

Map stats economy formulas were discussed:
- Pistol rounds are rounds 1 and 13.
- If pistol is won, round 2/14 is generally the buy-up and round 3/15 can be a bonus if the buy-up was won.
- If pistol is lost, round 2/14 is often a save/force context and round 3/15 is typically the full buy.
- Full buy after that depends on economy.
- Half buy threshold example discussed: rifle plus light shield is 3300 credits.
Future Riot data will determine exact economy classification where possible.

ASK COACH STATUS
Ask Coach is a dropdown chat below the nav, not a full modal.
It should overlay content and not push layout down.
Button is robot face near sync/profile controls.
It uses Supabase Edge Function and OpenAI when available, with local fallback.
Responses should be direct, human, and usually use clean bullet points.
Avoid markdown artifacts like bold markers when rendering.
It should have a thinking process:
1. Player feedback.
2. Player data.
3. Approved resources.
4. Structuring response.

Ask Coach should answer normal coaching questions in context, such as:
"I start strong for two games but the third is always a blowout. What can I do?"
